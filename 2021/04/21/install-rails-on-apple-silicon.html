在M1 Macbook Air上安装Rails记录

Apr 21, 2021

Env 基本情况:

ruby --version
ruby 2.6.3p62 (2019-04-16 revision 67580) [universal.arm64e-darwin20]
gem --version
3.0.3
which gem
/usr/bin/gem

使用rbenv将系统ruby版本切换到3.0.0

rbenv global 3.0.0

gem install rails
rails new blog

Errors 依次出现以下错误:

Solution 解决办法:

Notice the gem IS NOT /usr/bin/gem

which gem # /Users/jiayuran/.rbenv/shims/gem

gem uninstall \[package_name\]
gem install \[package_name\]

看起来一切如愿，接下来

cd blog
rails s

抛出错误

/Users/jiayuran/.rbenv/versions/3.0.0/lib/ruby/gems/3.0.0/gems/bootsnap-1.7.4/lib/bootsnap/load_path_cache/core_ext/kernel_require.rb:23:in `require': dlopen(/Users/jiayuran/.rbenv/versions/3.0.0/lib/ruby/gems/3.0.0/gems/nio4r-2.5.7/lib/nio4r_ext.bundle, 9): no suitable image found. 同样的方法，安装后就正常启动了。
